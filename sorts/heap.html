Summary

Heap Sort divides the array into 2 sections a heap section and a sorted section. The heap section uses a Max Heap. A Max heap is a data sructure in which each node has 2 children and is larger than both of it's children. This will build a tree structure. Heap Sort works by building a max heap in the array with the largest element in the 0 index. Once the heap is complete the element is removed and put into the "sorted" section by swapping it with the last unsorted element of the array. Then the element moved to the first position is processed to the correct location in the heap and the process starts over until all the elements have been moved to the sorted portion of the array.

Heap Sort works in 2 phases. In the first phase it builds the heap by starting at the nodes at the base of the tree and turning them into heaps. Then working it's way up to the first node and generating a single heap starting at the first node. The second phase repeatedly removes the first element, moves it to the sorted section, and then rebalances the heap.

Comparing Efficency Among Orderings

Heap Sort runs at a speed of O(n log(n)). However, it is important to note that unlike other sorts Heap Sort is actually less effective on an ordered list. This is because it needs to move the largest elements to the beginning of the list to build the heap.
